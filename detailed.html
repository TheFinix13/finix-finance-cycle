The problem

Every AI investing product sells one tool. The cycle is what matters.

Profit problem

Retail investors miss asymmetric opportunities through information overload and emotional trading.

Risk problem

Most investors are overexposed to one sector or hold losers too long. No dynamic hedging.

Income problem

Dividends are treated as afterthought, not as a core cash-flow engine.

Root cause — the silo problem: today's tools do not let trading logic, risk logic and income logic negotiate with each other. Every decision is made in isolation, and the losses live in the gaps.

The differentiator: agents don't run in sequence. Every proposal goes through Consensus — which weighs Profit, Risk and Income before allowing execution. "Good trade, bad portfolio" and "high yield, high risk" die at this layer.

The Consensus Engine

The negotiation loop.

StepWhat happens
1Trading Agent proposes: Buy AAPL · size · conviction · expected R
2Portfolio Agent stress-tests: sector exposure, VaR, drawdown vs high-water-mark. Approves, caps, or vetoes.
3Dividend Agent scores: yield-on-cost impact, dividend-cut probability, income-goal delta.
4News Agent attaches hard filters: earnings blackout, macro-release window, sentiment shock.
5Consensus Engine: Score = Profit − λ_risk·Risk + λ_income·IncomeDelta. Above threshold → execute. Below → send back for revision. Vetoed → journal + reason.

Learning loop: every executed and vetoed decision is a labelled event. Meta-RL updates λ weights nightly on paper / demo book before touching live.

Trading Agent · Live on demo

Find alpha. Under a validation gate.

Today: one validated zone-bounce strategy on EUR/USD, GBP/USD, USD/CAD. 7 concepts tested, 6 killed, 1 lives. Roughly 3–5 trades/week; 0.5–2% risk/trade; portfolio-wide 5% open-risk cap.

Methods: pattern strategies today · PPO / DQN under the validation pipeline · transformer + LSTM for short-horizon price/volume · sentiment from the News Agent.

Next-gen (R&D): M001 multi-agent squad — specialised strikers, a risk sentinel, and an aggregator that allocates trade slots. Adopted so far: aggregator variant ×1.3 throughput at higher per-trade conviction, no drawdown cost.

Dividend Agent · Scaffold + spec

Income, finally instrumented.

Today: package app/dividends/ exists inside the portfolio hub; KPIs defined before build — payout-realization rate + forward-yield accuracy.

Methods: dividend-safety classifier (XGBoost on payout ratio, FCF trend, coverage) · survival analysis for cut-probability · forward-yield optimisation under user risk caps · per-jurisdiction tax-impact estimator.

News Agent · To build (Phase 1)

The shared context layer.

Cross-cutting layer that reads market news, macro data, earnings, filings, and social sentiment — and feeds every other agent so no one decides blind.

What it does
  • Extracts structured events from unstructured news (earnings, guidance, M&A, macro releases)
  • Stores embeddings in a vector database for retrieval-augmented reasoning
  • Publishes hard event flags to Consensus (Fed day, earnings blackout, sanctions events)
  • Serves sentiment signals to Trading Agent as candidate features (validation-gated)
Under the hood
  • LangGraph orchestration around dual LLM providers (OpenAI + Anthropic)
  • Small-model default; escalate only on ambiguity
  • Aggressive caching + embeddings reuse
  • Provenance kept — every quoted event carries a citation

Who it's for

Retail

Individuals, families, busy professionals — one dashboard for every account, dividend planning, capital preservation.

SME / family office

Treasury and long-horizon capital — cycle discipline without an in-house desk. Multi-broker + multi-currency out of the box.

Institutional

Portfolio-intelligence API for brokers & fintechs; dedicated deployments for funds where in-house desks want the Consensus layer as infrastructure.

Wedge: Nigerian + Western broker coverage → diaspora investors incumbents ignore. Same platform scales up to institutional deployment because the architecture is the same — permissions differ.
